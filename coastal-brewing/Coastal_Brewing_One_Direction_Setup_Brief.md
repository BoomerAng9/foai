# Coastal Brewing One Direction Setup Brief

## Final answer

Use both systems, but give them one direction.

- **NVIDIA** is the low-cost model worker lane.
- **Feynman** is the research and verification lane.
- **ACHEEVY** is the only router.
- **AuditLedger** stores the receipts.
- **Chicken_Hawk** checks risk.
- **OpenClaw** executes approved work.
- **Boomer_Angs** operate the departments.

## One-direction diagram

```text
Owner intent
  -> ACHEEVY
    -> task packet
      -> Feynman for proof, supplier research, claims, market intelligence
      -> NVIDIA for drafts, classification, summarization, bulk ideation
      -> Owner/premium route for risk
    -> Boomer_Ang output
    -> Chicken_Hawk review
    -> AuditLedger receipt
    -> OpenClaw execution
```

## Why this solves the split

The previous two packages should not compete. The Feynman kit adds evidence. The NVIDIA kit adds cheap execution. The unified system needs both because a virtual coffee and tea company needs to move fast without making unsupported public claims.

## First setup steps

1. Unzip the v3 package.
2. Open it in Claude Code.
3. Paste `CREATE_WITH_CLAUDE_CODE.md`.
4. Configure `.env.local` from `configs/env.example`.
5. Run the smoke tests.
6. Import n8n workflows in manual mode.
7. Load agent prompts into OpenClaw.
8. Create Shopify/supplier accounts.
9. Run Feynman supplier and claim verification before publishing claims.
10. Use NVIDIA for drafts and low-risk internal work.

## First workflows to activate

1. Feynman Supplier Due Diligence.
2. Feynman Claim Verification Gate.
3. NVIDIA Bulk Content Generation.
4. NVIDIA Support Classifier.
5. Owner Approval Request.
6. Daily Operating Digest.

## Launch rule

Nothing public should claim organic, fair-trade, lab-tested, health, or certification status until the supplier proof exists, Feynman writes a receipt, Chicken_Hawk reviews it, and the owner approves the final wording.
