# Open Mind — Evaluation Rubric & Novelty Scoring v2.0

## Overview
The Evaluator stage scores output on independent dimensions. This separation is
load-bearing: CreativityPrism research proved that novelty and quality are often
negatively correlated. A solution can be correct but boring, or creative but broken.
Only dimension-separated scoring catches both failure modes.

---

## Novelty Assessment (0-6 Scale)

Adapted from the Cambridge Design Creativity rubric and CreativityPrism framework.

### Scoring Dimensions

**Originality (Q1)**: Does this solution use an approach that is uncommon for this problem type?
| Score | Description |
|-------|-------------|
| 0 | Exact copy of the most common approach |
| 1 | Minor variation on the standard approach |
| 2 | Recognizable pattern with one novel element |
| 3 | Mixed — some standard, some uncommon elements |
| 4 | Primarily uncommon approach with familiar elements |
| 5 | Novel approach that an expert would notice |
| 6 | Genuinely unprecedented — breaks new ground |

**Surprise (Q2)**: Would an expert find elements of this response surprising?
| Score | Description |
|-------|-------------|
| 0 | Entirely predictable — first thing an expert would suggest |
| 1 | Expected overall, one mildly interesting detail |
| 2 | Mostly expected, one unexpected connection |
| 3 | Several surprising elements within a familiar frame |
| 4 | Expert would pause and think "that's interesting" |
| 5 | Expert would want to discuss this with colleagues |
| 6 | Expert would reconsider their own assumptions |

**Non-Derivativeness (Q3)**: Does this go beyond the obvious first-thought solution?
| Score | Description |
|-------|-------------|
| 0 | This IS the first-thought solution |
| 1 | First thought with minor polish |
| 2 | Second-thought solution (first attempt refined) |
| 3 | Third-generation thinking — moved past initial instincts |
| 4 | Shows evidence of multiple restarts and reconceptions |
| 5 | Clearly the product of deep exploration |
| 6 | Could not have been reached without structured divergent process |

**Uniqueness (Q4)**: How much overlap with standard template responses?
| Score | Description |
|-------|-------------|
| 0 | Could be generated by any chatbot with default settings |
| 1 | 80%+ overlap with template responses |
| 2 | 60-80% overlap |
| 3 | 40-60% overlap — mix of template and original |
| 4 | 20-40% overlap — mostly original |
| 5 | <20% overlap — substantially unique |
| 6 | No recognizable template overlap |

## Quality Guard (0-6 Scale)

**Requirements Satisfaction (Q5)**: Does the solution correctly address core requirements?
| Score | Description |
|-------|-------------|
| 0-1 | Misses core requirements |
| 2-3 | Partially addresses requirements |
| 4-5 | Addresses all core requirements |
| 6 | Exceeds requirements with unexpected value |

**Coherence of Novel Elements (Q6)**: Is the novel element coherent, not random?
| Score | Description |
|-------|-------------|
| 0-1 | Novel element is disconnected / random |
| 2-3 | Novel element is loosely connected |
| 4-5 | Novel element is well-integrated |
| 6 | Novel element is essential — removing it breaks the solution |

**Practicality (Q7)**: Is this solution practically useful and implementable?
| Score | Description |
|-------|-------------|
| 0-1 | Theoretical only — cannot be implemented |
| 2-3 | Implementable with significant effort/unknowns |
| 4-5 | Clearly implementable with known technology |
| 6 | Implementation path is obvious and efficient |

---

## System Coherence (0-6 Scale)

**Internal Consistency (Q8)**: Is the solution internally consistent?
| Score | Description |
|-------|-------------|
| 0-2 | Contains contradictions or conflicting elements |
| 3-4 | Mostly consistent with minor tensions |
| 5-6 | Fully consistent — all parts reinforce each other |

**Integration Fitness (Q9)**: Does it integrate well with existing systems/constraints?
| Score | Description |
|-------|-------------|
| 0-2 | Requires major rework of existing systems |
| 3-4 | Moderate integration effort |
| 5-6 | Clean integration with existing stack |

## Composite Scoring Formula

```
overall_score = (0.3 × avg_quality) + (0.4 × avg_novelty) + (0.3 × avg_coherence)

where:
  avg_novelty   = mean(Q1, Q2, Q3, Q4)
  avg_quality   = mean(Q5, Q6, Q7)
  avg_coherence = mean(Q8, Q9)
```

### Decision Thresholds

| Condition | Action |
|-----------|--------|
| novelty avg < 3/6 | REJECT → return to Generator with "more creative" feedback |
| quality avg < 3/6 | REJECT → return to Generator with "fix correctness" feedback |
| coherence avg < 3/6 | REJECT → return to Generator with "improve integration" feedback |
| overall < 3/6 | REJECT → return to Planner for new approach |
| overall ≥ 3/6 AND all dimensions ≥ 3/6 | PASS → proceed to Verification Gates |
| Maximum 3 refinement loops | ESCALATE if still failing after 3 cycles |

## Binary Red Flags

Any "yes" triggers immediate regeneration — these are not scored, they are pass/fail.

- [ ] Does this closely match a known template or common pattern?
- [ ] Could this have been generated by simply following the most probable next tokens?
- [ ] Does this merely paraphrase the prompt or restate conventional wisdom?
- [ ] Is the "creativity" superficial (unusual words) rather than structural (novel approach)?
- [ ] Does this closely resemble any of the agent's previous outputs for similar tasks?

## NeoGauge Baseline Comparison

From ACL 2025: creativity should be state-aware. The same solution is less creative
if simpler alternatives exist.

**Process**:
1. The Planner explicitly states the "baseline obvious approach" in Stage 3
2. The Evaluator compares each option against this baseline
3. Novelty scores are relative to the baseline — not absolute

```
adjusted_novelty = raw_novelty × (1 - baseline_similarity)

where baseline_similarity = cosine_similarity(option_embedding, baseline_embedding)
```

If an option's adjusted novelty drops below 2/6 after baseline comparison, it is
effectively derivative regardless of raw scores.

## Evaluation Metrics for Automated Testing

| Metric | What It Measures | How to Compute | Threshold |
|--------|-----------------|----------------|-----------|
| Evidence Coverage Rate | % of key claims backed by sources | VERIFIED + PARTIALLY_SUPPORTED / total key claims | ≥ 0.85 (default), ≥ 0.95 (high-risk) |
| Citation Integrity | Citations actually support claims | Check cited snippet contains supporting statement | ≥ 0.90 |
| Retrieval Quality (RAG) | Context relevance and sufficiency | Context precision/recall, faithfulness (Ragas) | Task-dependent baseline |
| Faithfulness | Answer grounded in retrieved context | Ragas faithfulness or LLM-judge rubric | ≥ baseline + improvement |
| Option Diversity | Three options are meaningfully different | Rubric: distinct mechanisms + distinct tradeoffs | Pass/fail |
| Verification Compliance | Agent ran checklist and looped if failed | Transcript analysis: gates present? loops on failure? | ≥ 0.95 |
| Tool-Call Alignment | Tool calls match user intent | Prompt injection detection + alignment rules | ≥ 0.99 (high-risk) |
| Safety Leakage | No sensitive data appears without need | PII detector + leakage rubric | 0 critical leaks |
