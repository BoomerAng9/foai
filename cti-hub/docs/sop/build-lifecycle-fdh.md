# SOP: Build Lifecycle (FDH)

## Foster
- Capture intent, constraints, and success criteria.
- Identify text/voice/vision/data-source requirements.
- Generate ASCII preview for UI-facing work.
- Confirm scope before major implementation.

## Develop
- Implement approved scope via contracts, modules, and tests.
- Build in small vertical slices.
- Record change orders and preference updates in memory.
- Preserve current design-system direction.

## Hone
- Run QA/QC and policy/security checks.
- Validate acceptance criteria and docs completeness.
- Confirm no UI/design drift.

## Exit Criteria
- Foster: scope + preview aligned.
- Develop: slices implemented + tests present.
- Hone: checks pass + docs updated.
