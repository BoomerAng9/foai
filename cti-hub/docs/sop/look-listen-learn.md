# SOP: Look / Listen / Learn

## Look
- Route screenshots/images to vision-ready pathways.
- Extract layout, hierarchy, controls, and user intent.
- Log visual observations before implementation.

## Listen
- Treat prompt content as requirements.
- Reflect assumptions/constraints in structured form.
- Avoid coding before framing scope.

## Learn
- Store decisions, preferences, rejections, and change orders.
- Link memory to sessions, prototypes, and features.
- Use retained patterns for better future iteration.
