## Things to know before touching shield-policy

1. **YAML is the source of truth.** Never edit `generated/*.rs` by
   hand. Edit `config/shield/*.yml` and run
   `python scripts/compile-shield-policy.py`. CI's drift-gate
   (`--check`) enforces this.

2. **Every module follows the pure-predicate pattern.** Types +
   pure functions + runtime tests + Kani proofs. Stateful logic
   lives in the owning Hawk's external runtime. Don't add state to
   the crate.

3. **Kani harnesses cover 54 invariants across 22 YAML
   `kani_properties` entries.** Any new module should ship its
   harnesses in the same PR. The house rule is "no new module
   without proofs."

4. **Cloud Build is the CI.** Never add `.github/workflows/*.yml`
   for new pipelines — Cloud Build via `cloudbuild.yaml` is the
   foai monorepo standard (memory: `feedback_ci_cd_is_cloud_build`).

5. **`verify-substrates.sh` is required.** All 3 substrates must
   compile clean. A dependency bump that breaks WASM or ARM64
   fails CI immediately.

6. **The Paranoia invariant is the marquee proof.** `kani_paranoia_
   refuses_crypt_ang` proves that for ALL invocations where
   `hawk == LilDoubtHawk` AND `commander == CryptAng`, the validator
   returns `Err(ProhibitedCommander)`. If this proof breaks, the
   independent-auditor reporting line is compromised. Watch it
   carefully when touching commander prohibition code.
