# User Journey Design — Discovery Checklist

Source: Notion `User Journey` doc, ingested 2026-04-08.
Use: ACHEEVY's Guide Me path uses this for new aiPLUG design sessions. 20 "How might we" questions to walk a user through journey design.

---

1. How might we reduce friction and simplify the user journey for maximum efficiency and satisfaction?
2. How might we use data and analytics to optimize the user journey over time?
3. How might we design a user journey that's scalable and adaptable as our product evolves?
4. How might we create a user journey that's personalized for each user's needs and preferences?
5. How might we use microinteractions and other design elements to enhance the user journey?
6. How might we design a user journey that's accessible for users with disabilities?
7. How might we create a user journey that's easy to understand and use, even for non-technical users?
8. How might we use the user journey to build trust and credibility with our users?
9. How might we design a user journey that's flexible and adaptable to different user contexts and scenarios?
10. How might we use the user journey to provide real-time support and guidance to users?
11. How might we create a user journey that's consistent across all devices and platforms?
12. How might we use the user journey to encourage users to explore and discover new features and functionality?
13. How might we design a user journey that's easy to update and maintain?
14. How might we use the user journey to align with our brand values and mission?
15. How might we use the user journey to create a positive, inclusive user experience for all users?
16. How might we use the user journey to increase the reach and impact of our product?
17. How might we design a user journey that's sensitive to user privacy and security concerns?
18. How might we use the user journey to create a more efficient and streamlined user experience for all users?
19. How might we use the user journey to stay ahead of industry trends and best practices in UX design?
20. How might we use the user journey to foster a sense of community and connection among our users?

---

## How ACHEEVY uses this

In the **Guide Me path** (per `project_acheevy_guide_me_manage_paths.md`), when a user is designing a new aiPLUG or workflow, ACHEEVY walks them through these 20 questions one at a time. Each answer feeds into the design brief that gets committed via `pmo.commission()` before the build agents pick it up.

In the **Manage path**, ACHEEVY answers them itself based on best practices + competitor benchmarks + the user's existing context, then presents the design brief for confirmation.

Both paths produce the same output structure — a comprehensive user journey map that the build agents (Boomer_CDO, Boomer_CTO, design tools per `project_design_routing.md`) can execute against.
